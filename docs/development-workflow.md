## Development Lifecycle & Workflow
To simulate a production-grade environment, this repository adheres to the following standards:
1. **Planning:** Problem decomposition using GitHub Projects cards.
2. **Isolation:** Strict feature-branch workflow (`feat/`, `fix/`, `chore/`).
3. **Communication:** Atomic commits using the Conventional Commits standard.
4. **Review:** "Squash and Merge" via Pull Requests to maintain a linear history.
5. **Deployment:** Semantic versioning tags for release management.

* **Commit message convention:** `<type(scope): description>`
* **Branch name convention:** `<type/short-description-kebab-case>`
